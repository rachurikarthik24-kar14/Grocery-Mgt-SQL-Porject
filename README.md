# Grocery-Mgt-SQL-Porject

The Grocery Store Management System is a MySQL-based database project developed to manage and analyze grocery store operations efficiently. 
The database consists of seven interconnected tables: Customers, Products, Categories, Suppliers, Employees, Orders, and Order Details. 
These tables are linked using primary and foreign key relationships to maintain data integrity and support real-world business operations.

The project focuses on analyzing customer purchasing behavior, product performance, supplier contribution, employee productivity, and sales trends using SQL queries. Various SQL concepts such as Joins, Aggregate Functions, Group By, Order By, Subqueries, and Filtering techniques were used to extract meaningful business insights from the data.

Key analyses performed include identifying top customers, best-selling products, revenue-generating products, supplier contributions, employee performance, and order trends. The insights generated help businesses improve inventory management, monitor sales performance, optimize supplier relationships, and make data-driven decisions.

This project demonstrates the practical application of MySQL in designing relational databases, managing retail business data, and transforming raw transactional data into actionable business intelligence
